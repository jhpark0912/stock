# 애플리케이션 로컬 배포 가이드

개발 경험이 없는 사용자를 위해 로컬 환경에서 백엔드 서버와 프론트엔드 앱을 실행하는 방법을 안내합니다.

## 1. 사전 준비

애플리케이션을 실행하려면 아래 두 가지 프로그램을 먼저 설치해야 합니다.

### 1.1. Node.js (프론트엔드용)

- **설치 방법:**
  1. [Node.js 공식 홈페이지](https://nodejs.org/ko/)에 접속합니다.
  2. **LTS (Long Term Support)** 버전을 다운로드하여 설치합니다. (설치 과정에서 별도 설정을 변경할 필요 없이 'Next'만 눌러 진행하면 됩니다.)
- **설치 확인:**
  - `Win + R` 키를 눌러 실행창을 열고 `cmd`를 입력하여 명령 프롬프트를 엽니다.
  - 아래 명령어를 입력하여 버전 번호가 표시되는지 확인합니다.
    ```bash
    node -v
    npm -v
    ```

### 1.2. Python (백엔드용)

- **설치 방법:**
  1. [Python 공식 홈페이지](https://www.python.org/downloads/)에 접속합니다.
  2. 최신 버전을 다운로드하여 설치합니다.
  3. **중요:** 설치 시작 화면에서 **"Add Python to PATH"** 체크박스를 반드시 선택하고 설치를 진행합니다.
- **설치 확인:**
  - 명령 프롬프트에서 아래 명령어를 입력하여 버전 번호가 표시되는지 확인합니다.
    ```bash
    python --version
    ```

## 2. 백엔드 배포

백엔드 API 서버를 실행하는 단계입니다.

1.  **백엔드 폴더로 이동**
    - 전달받은 프로젝트 폴더 내의 `backend` 폴더로 들어갑니다.

2.  **필요한 라이브러리 설치**
    - 해당 폴더에서 명령 프롬프트를 엽니다. (폴더 주소창에 `cmd` 입력 후 엔터)
    - 아래 명령어를 실행하여 서버에 필요한 라이브러리들을 설치합니다.
      ```bash
      pip install -r requirements.txt
      ```

3.  **환경 변수 설정**
    - `backend` 폴더에 있는 `.env.example` 파일의 복사본을 만듭니다.
    - 파일 이름을 `.env`로 변경합니다.
    - 메모장으로 `.env` 파일을 열어 `GEMINI_API_KEY=` 뒷부분에 발급받은 Gemini API 키를 붙여넣고 저장합니다.

4.  **백엔드 서버 실행**
    - 명령 프롬프트에서 아래 명령어를 실행합니다.
      ```bash
      uvicorn app.main:app --host 0.0.0.0 --port 8000
      ```
    - `Application startup complete` 메시지가 보이면 서버가 성공적으로 실행된 것입니다.
    - **이 명령 프롬프트 창은 닫지 말고 그대로 두어야 합니다.** (서버를 종료하려면 `Ctrl + C`를 누릅니다.)

## 3. 프론트엔드 배포

사용자 화면을 담당하는 프론트엔드 애플리케이션을 실행하는 단계입니다.

1.  **프론트엔드 폴더로 이동**
    - 전달받은 프로젝트 폴더 내의 `frontend` 폴더로 들어갑니다.

2.  **필요한 라이브러리 설치**
    - 해당 폴더에서 **새로운 명령 프롬프트**를 엽니다.
    - 아래 명령어를 실행하여 필요한 라이브러리들을 설치합니다.
      ```bash
      npm install
      ```

3.  **애플리케이션 빌드**
    - 라이브러리 설치가 완료되면, 아래 명령어를 실행하여 배포용 파일을 생성합니다.
    - 이 과정은 몇 분 정도 소요될 수 있습니다.
      ```bash
      npm run build
      ```
    - 완료되면 `frontend` 폴더 내에 `dist`라는 새로운 폴더가 생성됩니다.

4.  **로컬 서버 프로그램 설치**
    - 빌드된 결과물을 웹사이트처럼 띄워줄 간단한 서버 프로그램을 설치합니다.
      ```bash
      npm install -g serve
      ```

5.  **프론트엔드 서버 실행**
    - 아래 명령어를 실행하여 프론트엔드 서버를 시작합니다.
      ```bash
      serve -s dist -l 3000
      ```
    - `Accepting connections at http://localhost:3000` 메시지가 표시되면 성공입니다.

## 4. 애플리케이션 접속

1.  백엔드와 프론트엔드 서버가 모두 실행 중인 상태인지 확인합니다. (명령 프롬프트 창 2개가 모두 켜져 있어야 합니다.)
2.  웹 브라우저(Chrome, Edge 등)를 열고 주소창에 아래 주소를 입력합니다.
    - **http://localhost:3000**

## ※ 배포 파일 목록

개발자가 비개발자에게 애플리케이션을 전달할 때, 아래 목록의 파일과 폴더 전체를 하나의 폴더에 담아 압축 파일(`.zip`)로 전달하면 됩니다.

### 포함할 항목

- `backend/`
- `frontend/`
- `.gitignore`
- `README.md`
- `DEPLOYMENT.md` (현재 이 파일)

### 제외할 항목 (전달할 필요 없는 항목)

- `node_modules/`
- `__pycache__/`
- `.git/`
- `.vscode/`, `.idea/` 등 개인 개발 환경 설정 폴더
- `.env` (API 키 등 민감 정보가 포함되어 있으므로, `.env.example`만 포함)
- 임시 폴더 (예: `tmpclaude-*`)
