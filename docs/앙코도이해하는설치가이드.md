## 1. 시작하기 전에

### 📌 이 프로그램이 필요한 것

이 프로그램은 두 부분으로 이루어져 있습니다:

1. **백엔드 (Backend)**: 주식 데이터를 가져오고 처리하는 서버
2. **프론트엔드 (Frontend)**: 사용자가 보는 웹 화면

두 프로그램을 모두 실행해야 정상적으로 작동합니다.

### ⚠️ 주의사항

- 인터넷 연결이 필요합니다 (주식 데이터를 온라인에서 가져옵니다)
- Gemini AI API 키가 필요합니다 (AI 분석 기능을 사용하려면)
- 설치 시간은 약 20-30분 정도 소요됩니다

## 2. 필수 프로그램 설치

### 2-1. Python 설치 (백엔드용)

#### ✅ Python이 이미 설치되어 있는지 확인

1. Windows 검색창에서 `cmd` 또는 `명령 프롬프트` 검색
2. 명령 프롬프트를 열고 다음 명령어 입력:

```bash
python --version
```

3. 결과 확인:
   - `Python 3.8` 이상 버전이 표시되면 ✅ **이미 설치됨** → 2-2로 이동
   - `'python'은(는) 내부 또는 외부 명령...` 오류가 나면 ❌ **설치 필요** → 아래 단계 진행

#### 📥 Python 설치하기

1. Python 공식 웹사이트 방문: https://www.python.org/downloads/
2. 노란색 "Download Python" 버튼 클릭
3. 다운로드된 설치 파일 실행
4. ⚠️ **중요**: 설치 시작 화면에서 **"Add Python to PATH"** 체크박스 반드시 선택!
5. "Install Now" 클릭
6. 설치 완료 후 명령 프롬프트를 다시 열고 `python --version` 입력하여 확인

---

### 2-2. Node.js 설치 (프론트엔드용)

#### ✅ Node.js가 이미 설치되어 있는지 확인

명령 프롬프트에서:

```bash
node --version
```

- `v18` 이상 버전이 표시되면 ✅ **이미 설치됨** → 3장으로 이동
- 오류가 나면 ❌ **설치 필요** → 아래 단계 진행

#### 📥 Node.js 설치하기

1. Node.js 공식 웹사이트 방문: https://nodejs.org/
2. "LTS" (추천) 버전 다운로드 클릭
3. 다운로드된 설치 파일 실행
4. 기본 설정대로 "Next" 클릭하며 설치 진행
5. 설치 완료 후 명령 프롬프트를 다시 열고 `node --version` 입력하여 확인

## 3. 프로젝트 파일 준비

### 3-1. 압축 파일 다운로드 및 압축 해제

1. 제공받은 `stock_deployment_XXXXXX.zip` 파일을 준비합니다
2. 파일을 원하는 폴더에 압축 해제합니다
   - 예: `C:\stock` 또는 `D:\projects\stock`
3. 압축 해제된 폴더 구조 확인:

```
stock/
├── backend/        (백엔드 폴더)
├── frontend/       (프론트엔드 폴더)
├── .env.example    (환경 변수 템플릿)
└── README.md
```

### 3-2. 환경 변수 파일 설정

#### 백엔드 환경 변수

1. `backend` 폴더로 이동
2. `.env.example` 파일을 복사하여 `.env` 파일 생성:
   - Windows 탐색기에서: `.env.example` 파일 복사 → 붙여넣기 → 이름을 `.env`로 변경
3. `.env` 파일을 메모장으로 열기
4. 다음 내용 수정:

```env
# Gemini API Key (필수 - AI 분석 기능)
GEMINI_API_KEY=여기에_발급받은_API_키_입력

# Environment
ENVIRONMENT=development

# CORS Origins
ALLOWED_ORIGINS=http://localhost:5173,http://localhost:3000

# Server Configuration
HOST=127.0.0.1
PORT=8080
```

**🔑 Gemini API 키 발급 방법:**
1. https://aistudio.google.com/apikey 방문
2. Google 계정으로 로그인
3. "Create API Key" 버튼 클릭
4. 생성된 키를 복사하여 `.env` 파일의 `GEMINI_API_KEY=` 뒤에 붙여넣기

#### 프론트엔드 환경 변수

1. `frontend` 폴더로 이동
2. `.env.example` 파일을 복사하여 `.env` 파일 생성
3. `.env` 파일을 메모장으로 열기
4. 내용 확인 (기본값 사용):

```env
VITE_API_URL=http://localhost:8080
```

## 4. 백엔드 서버 설정 및 실행

### 4-1. 명령 프롬프트 열기

1. Windows 검색창에서 `cmd` 검색
2. 명령 프롬프트 열기

### 4-2. 백엔드 폴더로 이동

```bash
cd C:\stock\backend
```

> 💡 **팁**: 실제 압축 해제한 경로로 변경하세요.
> 
> 예시:
> - `cd C:\Users\사용자이름\Downloads\stock\backend`
> - `cd D:\projects\stock\backend`

### 4-3. Python 가상환경 생성 (선택사항이지만 권장)

```bash
python -m venv venv
```

> 📝 **가상환경이란?**: 이 프로젝트만을 위한 독립적인 Python 환경을 만듭니다.
> 다른 프로젝트와 충돌을 방지합니다.

### 4-4. 가상환경 활성화

```bash
venv\Scripts\activate
```

> ✅ 성공하면 명령 프롬프트 앞에 `(venv)`가 표시됩니다:
> ```
> (venv) C:\stock\backend>
> ```

### 4-5. 필요한 라이브러리 설치

```bash
pip install -r requirements.txt
```

> ⏱️ 설치 시간: 약 2-5분 (인터넷 속도에 따라 다름)
> 
> 화면에 여러 줄의 설치 메시지가 나타나며, 마지막에 "Successfully installed ..." 메시지가 표시되면 성공입니다.

### 4-6. 백엔드 서버 실행

```bash
python -m uvicorn app.main:app --host 127.0.0.1 --port 8080 --reload
```

> ✅ 성공 메시지:
> ```
> INFO:     Uvicorn running on http://127.0.0.1:8080 (Press CTRL+C to quit)
> INFO:     Started reloader process [XXXXX] using WatchFiles
> INFO:     Started server process [XXXXX]
> INFO:     Waiting for application startup.
> INFO:     Application startup complete.
> ```

> ⚠️ **이 명령 프롬프트 창을 닫지 마세요!** 백엔드 서버가 실행 중인 상태로 유지해야 합니다.

## 5. 프론트엔드 설정 및 실행

### 5-1. 새 명령 프롬프트 창 열기

백엔드 서버를 실행한 명령 프롬프트는 그대로 두고, **새로운 명령 프롬프트 창**을 엽니다.

### 5-2. 프론트엔드 폴더로 이동

```bash
cd C:\stock\frontend
```

> 💡 실제 압축 해제한 경로로 변경하세요.

### 5-3. Node.js 패키지 설치

```bash
npm install
```

> ⏱️ 설치 시간: 약 3-10분 (인터넷 속도에 따라 다름)
> 
> 많은 패키지를 설치하므로 시간이 걸립니다. 설치가 완료될 때까지 기다려주세요.
> 
> ⚠️ 경고 메시지(`WARN`)는 무시해도 괜찮습니다. 에러(`ERR!`)만 주의하세요.

### 5-4. 프론트엔드 개발 서버 실행

```bash
npm run dev
```

> ✅ 성공 메시지:
> ```
> VITE v7.x.x  ready in XXX ms
> 
> ➜  Local:   http://localhost:5173/
> ➜  Network: use --host to expose
> ➜  press h + enter to show help
> ```

> ⚠️ **이 명령 프롬프트 창도 닫지 마세요!** 프론트엔드 서버가 실행 중인 상태로 유지해야 합니다.

## 7. 문제 해결

### 🚨 백엔드 서버가 실행되지 않는 경우

#### 문제 1: "python을 찾을 수 없습니다"

**해결 방법:**
1. Python 설치 확인
2. 환경 변수 PATH에 Python 경로 추가
3. 명령 프롬프트 재시작

#### 문제 2: "포트 8080이 이미 사용 중"

**해결 방법:**
포트 번호를 변경합니다:

```bash
python -m uvicorn app.main:app --host 127.0.0.1 --port 8081 --reload
```

그리고 `frontend/.env` 파일도 수정:
```env
VITE_API_URL=http://localhost:8081
```

#### 문제 3: "ModuleNotFoundError"

**해결 방법:**
```bash
pip install -r requirements.txt
```
다시 실행하여 누락된 패키지 설치

---

### 🚨 프론트엔드가 실행되지 않는 경우

#### 문제 1: "node를 찾을 수 없습니다"

**해결 방법:**
1. Node.js 설치 확인
2. 명령 프롬프트 재시작

#### 문제 2: "포트 5173이 이미 사용 중"

**해결 방법:**
자동으로 다른 포트(예: 5174)를 사용합니다. 화면에 표시된 포트 번호로 접속하세요.

#### 문제 3: "npm install 중 에러"

**해결 방법:**
```bash
# npm 캐시 정리
npm cache clean --force

# 다시 설치
npm install
```

---

### 🚨 웹 페이지에서 데이터가 로딩되지 않는 경우

#### 체크리스트:

1. ✅ 백엔드 서버가 실행 중인가?
   - 명령 프롬프트에 "Uvicorn running..." 메시지 확인

2. ✅ 프론트엔드 서버가 실행 중인가?
   - 명령 프롬프트에 "VITE ... ready" 메시지 확인

3. ✅ `.env` 파일 설정이 올바른가?
   - `backend/.env`: GEMINI_API_KEY 설정됨
   - `frontend/.env`: VITE_API_URL=http://localhost:8080

4. ✅ 방화벽이 차단하고 있지 않은가?
   - Windows Defender 방화벽 → "앱이 방화벽을 통과하도록 허용" → Python 허용

5. ✅ 인터넷 연결이 되어 있는가?

## 8. 프로그램 종료하기

### 8-1. 정상 종료 방법

1. **프론트엔드 서버 종료**:
   - 프론트엔드 명령 프롬프트 창에서
   - `Ctrl + C` 키 누르기
   - `작업을 중지하시겠습니까? (Y/N)` → `Y` 입력 후 Enter

2. **백엔드 서버 종료**:
   - 백엔드 명령 프롬프트 창에서
   - `Ctrl + C` 키 누르기
   - `작업을 중지하시겠습니까? (Y/N)` → `Y` 입력 후 Enter

3. **가상환경 비활성화** (선택사항):
   ```bash
   deactivate
   ```

### 8-2. 다음에 다시 실행하기

다음에 프로그램을 다시 실행할 때는 2장(필수 프로그램 설치)은 건너뛰고 바로 4장부터 시작하면 됩니다:

1. 명령 프롬프트 2개 열기
2. 첫 번째: 백엔드 실행
   ```bash
   cd C:\stock\backend
   venv\Scripts\activate
   python -m uvicorn app.main:app --host 127.0.0.1 --port 8080 --reload
   ```
3. 두 번째: 프론트엔드 실행
   ```bash
   cd C:\stock\frontend
   npm run dev
   ```
4. 브라우저에서 `http://localhost:5173` 접속
